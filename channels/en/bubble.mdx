The **Chat Bubble** lets you connect your Agent directly to users on your website or app. You can customize its look, define its behavior, and embed it with a simple script.

When you open the Bubble channel settings (**Channels > Bubble**), you'll see a **two-panel layout**:

- **Left panel** — configuration tabs where you adjust content, appearance, and embedding.
- **Right panel** — a live preview that updates in real time as you make changes.

Use the tabs below to explore configuration options:

<Tabs>
  <Tab title="Content" icon="text">
    ## Content

    Configure the messages and text elements that appear inside your chat bubble.  
    Navigate to **Channels > Bubble > Content** to adjust these settings.

    ### Welcome message

    Set the first message your Agent displays when a user opens the chat. You can enable or disable the welcome message with a toggle.

    ### Suggested messages

    Predefine up to **5 quick-start suggestions** (each up to 100 characters) to help users engage faster. These appear as clickable pills below the welcome message. Toggle suggested messages on or off as needed.

    ### Dismissible notice

    Add an informational notice that appears at the bottom of the chat window (above the input field). This is useful for privacy disclaimers or terms-of-service reminders.

    - Edit the notice text using the **rich text editor** — supports **bold**, *italic*, <u>underline</u>, ~~strikethrough~~, and links.
    - If you leave the notice empty, a default privacy notice is shown automatically.
    - If you explicitly clear the notice, nothing is displayed.

    <Tip>
    Use the dismissible notice to inform users that conversations are processed by AI, linking to your privacy policy for transparency.
    </Tip>

    The notice is automatically dismissed when the user sends their first message or clicks a suggested message pill.

    ### Footer text

    Customize the text displayed at the very bottom of the chat panel. Use the **rich text editor** to add formatted text and links (e.g., "Powered by [Your Brand]").

    ### Feedback collection

    Toggle whether users can provide feedback on Agent responses. *(Coming soon)*

    <Note>
    Preview updates in real time with the chat bubble widget shown on the right side of the Console.
    </Note>
  </Tab>

  <Tab title="Appearance" icon="palette">
    ## Appearance

    Personalize the look and feel of your chat bubble so it matches your brand identity.  
    Navigate to **Channels > Bubble > Appearance** to adjust these settings.

    ![bubble-appearance.png](/images/channels/bubble/en/bubble-appearance.png)

    ### Header

    #### Display name

    Toggle **Show display name** to control whether a name appears in the chat header. By default, your organization name is used. You can override it by entering a **custom header title** — this is the name users see at the top of the chat window.

    #### Icon vs Logo mode

    Use the **Prefer logo over icon** toggle to switch between two header image modes:

    - **Off (Icon mode, default)** — Upload a square image displayed as a small **round icon** (32×32 circle) in the chat header.
    - **On (Logo mode)** — Upload a wider **title image / logo** displayed in rectangular format in the header.

    <Note>
    When **Logo mode** is enabled, you must upload a logo image before you can save. If no logo is uploaded, saving is blocked and you'll see an error message.
    </Note>

    #### Header color

    Choose the header background color:
    - **Solid color:** Select from the color picker.
    - **Gradient:** Enable gradient mode, pick a secondary color, and choose **Horizontal** or **Vertical** direction.

    ### Bubble trigger

    Customize the floating button that users click to open the chat:
    - **Bubble color** — the background color of the trigger pill/icon.
    - **Bubble text** — the label shown on the pill trigger (max 30 characters).
    - **Alignment** — position the bubble on the **left** or **right** side of the screen.

    <Note>
    Preview updates in real-time with the chat bubble widget shown on the right side of the Console.
    </Note>
  </Tab>

  <Tab title="Embed" icon="code">
    ## Embed the Chat Bubble

    To integrate the chat bubble into your website or app, copy the provided script snippets.  
    You can find them under **Channels > Bubble > Embed**.

    ![bubble-embed.png](/images/channels/bubble/en/bubble-embed.png)

    ### 1. Add to the `<head>` of your site:
    ```html
    <script src="https://consola.semantiks.com/bubble-chat.js"></script>
    ```

    ### 2. Add to the `<body>` of your site:
    ```html
    <script>
      document.addEventListener('DOMContentLoaded', () => {
        window.BubbleChat.initWidget({
          agent_id: 'your-agent-id',
          position: 'bottom_right',
          bubble_theme: 'simple_light',
          api_key: 'Your API KEY here',
        });
      });
    </script>
    ```

    ### 3. (Optional) Pass user information
    ```javascript
    window.BubbleChat.initWidget({
      agent_id: 'your-agent-id',
      position: 'bottom_right',
      bubble_theme: 'simple_light',
      api_key: 'Your API KEY here',
      user: {
        user_id: 'user_123',
        profile_name: 'John Doe',
        user_metadata: {
          plan: 'premium',
          company: 'Acme Inc'
        }
      }
    });
    ```

    <Note>
    If no user information is provided, the widget generates a guest user ID automatically and persists it across visits using the browser's local storage.
    </Note>

    <Tip>
    Use the **Embed** tab in the Console to copy pre-filled snippets with your agent's actual ID and API key — no manual editing required.
    </Tip>
  </Tab>
</Tabs>

## Conversation behavior

The chat bubble supports several message display modes:

- **Writing** — messages appear with a typewriter animation.
- **Streaming** — messages stream in as the Agent generates them.
- **Dots** — a thinking indicator (animated dots) is shown while the Agent processes.

Users can restart a conversation at any time using the refresh button in the chat header. This clears the current thread and starts a new session.

<Tip>
  Always test your bubble configuration using the live preview before saving. Once saved, the changes are reflected immediately on your website.
</Tip>
