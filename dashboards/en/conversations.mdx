The Semantiks Console dashboard gives enterprises real‑time visibility into conversations led by AI agents across all connected channels. Its design supports two complementary objectives: understand system‑level behavior (volumes, duration, engagement) and quickly drill into any single session for evidence‑based decisions (retuning prompts, updating knowledge, or handing issues to operations). Start with the high‑level structure to frame what you're seeing, then use the right‑side panel to read details without losing your place.

<Note>

Additionally, our team is currently working on fixing certain reporting bugs affecting KPIs to improve accuracy and reliability.
</Note>

## Overview of the Dashboard

When you access the **Conversations Dashboard**, you will see:

- **KPIs (Key Performance Indicators):** High-level metrics summarizing performance.
- **Conversations Table:** A detailed list of all conversations with users.
- **Conversation Panel:** A right-side panel that shows more details when you click on a conversation.

---

## KPIs Cards

The top section provides at-a-glance metrics:

- **Total conversations** – The number of conversations in the selected time range.
- **Average duration** – The average length of conversations.
- **User messages** – Average number of messages sent by users.
- **Agent messages** – Average number of messages sent by the agent.

<Note>Green arrows indicate growth compared to the previous period, while red arrows indicate a decrease.</Note>
<Tip>You can filter the time range by selecting **Today, 7D, 30D, or Year**.</Tip>
---

## Conversations Table

Below the KPIs, you will find a table with all recorded conversations. Each row represents a conversation with details such as:

- **Status:** Current status of the conversation (Active, Ended, or Not Started).
- **Start:** Date and time when the conversation started.
- **Channel:** The communication channel used (e.g., Freshchat Bubble, WhatsApp).
- **Tools:** Tools or skills used by the agent during the conversation.
- **Control:** Indicates who handled the conversation. For AI-handled conversations, a generic label is shown (e.g., "AI"). For human-handled conversations, the **name of the specialist** attending is displayed.
- **Profile Name:** Name of the user (if captured by the channel).
- **External ID:** User ID from the channel.
- **Number of Messages:** Total messages exchanged (with the AI agent).
- **First Response Time (FRT):** The time elapsed between the user's first message and the agent's first reply. Displayed in a human-readable format (e.g., `45s`, `2m 30s`, `1h 15m`).
- **Average Response Time (ART):** The average time the agent takes to respond across all turns in the conversation.

<Tip>
Click on any row to expand the conversation panel on the right.
</Tip>

## Conversation Panel

When you select a conversation from the table, a detailed panel opens on the right side. The panel has two tabs: **Chat** and **Details**.

### Chat tab

The Chat tab shows the full conversation transcript. Messages are displayed chronologically, with visual cues that differentiate senders:

- **AI Agent** messages appear on the left with an "AI" avatar.
- **Customer** messages appear on the right.
- **Specialist** (human agent) messages are also displayed on the left, labeled with the specialist's actual name when available.
- **System events** (such as escalations or handoffs) appear as centered, muted labels.

#### Tool execution cards

When the AI agent executes a tool during a conversation, a **tool card** is interleaved in the timeline at the exact moment the tool was invoked. Each card shows:

- **Tool icon** — the tool's custom icon if one is configured, or a default wrench icon.
- **Tool name** — the name of the tool that was executed.
- **"Tool" tag** — a small label identifying the card as a tool invocation.
- **Result** — the output or message returned by the tool (truncated if very long).
- **Timestamp** — the exact time (with seconds) the tool was called.

If the tool execution failed, the card is styled with an error indicator so you can quickly spot problems.

<Tip>
Tool cards let you see exactly what your agent did "behind the scenes" during the conversation, making it easy to debug unexpected responses or verify that tools are working correctly.
</Tip>

### Details tab

The Details tab provides structured metadata about the conversation, organized in collapsible groups:

#### Identification
- **ID** — The unique conversation identifier.
- **Client** — The external user ID from the channel.
- **Profile** — The user's profile name as captured by the channel (e.g., their WhatsApp display name).
- **Channel** — The integration and channel type used.

#### Metrics
- **Messages** — Total number of messages exchanged.
- **First Response Time (FRT)** — Time until the agent's first reply.
- **Average Response Time (ART)** — Average reply time across the conversation.
- **Flag** — Conversation quality flag (Good, Bad, or Needs Attention), shown as a color-coded badge.
- **Sentiment** — Detected sentiment tags for the conversation.
- **Tools** — A list of all tools the agent used during the conversation, each displayed with its icon and name.
- **Last Control** — Who last handled the conversation: either "AI Agent" or the **specialist's name** (resolved from the team member who was assigned).

#### Metadata

The metadata section shows additional user metadata passed from the integrated channel (e.g., custom attributes, handoff data). The section handles three states:

- **Loading** — While metadata is being fetched, a loading indicator is shown.
- **No metadata** — If no metadata is available for the user, an informational message is displayed.
- **Error** — If metadata could not be loaded, an error message appears.
- **Metadata present** — When metadata exists, it is displayed as syntax-highlighted JSON.

<Note>
Metadata availability depends on what your integrated channels send. Some channels may not pass any custom user attributes.
</Note>

## Common Actions

- **Search conversations** — Use the search bar to filter conversations by keyword.
- **Filter by date range** — Narrow results to a specific time period.
- **Export** — Download conversation data for offline analysis.
- **Navigate to related views** — From any conversation, you can jump to the user's contact record or the relevant evaluation.
