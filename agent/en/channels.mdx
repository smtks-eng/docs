## Connect your agent to real users

The **Connect** page is organized into two tabs:

- **Channels** — connect and manage the channels where your agent talks to users.
- **Messaging Behaviour** — configure how your agent handles messages on each connected channel.

This is the step that turns your agent from "configured" into "live".

---

## Channels tab

The Channels tab is organized in 3 sections:

- **Channels**: publish your agent in owned and external channels (Web Bubble, External Chat, WhatsApp, and others)
- **Integrations**: connect external tools/services used by your operation
- **API**: send and receive messages from your own systems

### External Chat (fastest launch)

**External Chat** gives you a shareable URL you can send in seconds.

<Steps>
  <Step title="Go to Connect > Channels">
    Open your agent and navigate to the Connect page.
  </Step>
  <Step title="Open External Chat">
    Select the External Chat channel card.
  </Step>
  <Step title="Copy the public URL">
    Click the share option and copy the generated link.
  </Step>
  <Step title="Test as an end user">
    Open the link in an incognito window and run a few real use-case questions.
  </Step>
</Steps>

### Web Bubble (embed in your website)

Use **Web Bubble** when you want the agent inside your own site.

<Steps>
  <Step title="Open Bubble channel">
    In Connect > Channels, open the Bubble card.
  </Step>
  <Step title="Generate credentials/snippets">
    The Console provides the snippet and keys needed for embedding.
  </Step>
  <Step title="Add snippet to your website">
    Paste the script where your implementation team manages site code.
  </Step>
  <Step title="Customize appearance">
    Configure brand settings like launcher style and welcome behavior.
  </Step>
</Steps>

### Meta channels (WhatsApp, Messenger, Instagram)

The Connect page always displays a **Meta** section with three channel cards:

| Channel | Description |
|---|---|
| **WhatsApp** | WhatsApp Business channel via Meta |
| **Messenger** | Facebook Messenger channel via Meta |
| **Instagram** | Instagram messaging channel via Meta |

Each card shows a **Connect** button that opens the setup flow for that specific channel. You do not need to configure the Meta integration first — the cards are always visible so you can start the connection process at any time.

Once you have connected one or more accounts for a channel type, a **Channels (n)** button appears on the card showing the count of active connections. Click it to view, edit, or reconnect your existing connections.

## Integrations

The **Integrations** section shows third-party platforms your agent can connect to. Each integration card has a **Configure** button to set up the required credentials. Once configured, most integrations also show a **Channels** button that lets you manage the individual channels connected through that integration.

### Intercom integration

Intercom uses a **workspace-based** connection model. The setup is split into two parts:

#### 1. Configure the integration

Click **Configure** on the Intercom card and provide:

| Field | Description |
|---|---|
| **Client Secret** | The client secret from your Intercom private app. |
| **Admin ID** | The ID of the Intercom admin that will be associated with the integration. |

<Note>
  You can find the Admin ID in your Intercom Developer Hub. It identifies which admin account the agent operates under.
</Note>

#### 2. Manage workspaces

Once the Intercom integration is configured, a **Workspaces** button appears on the card. Click it to open the Intercom Workspaces panel, where you can:

- **View** all currently connected Intercom workspaces and their status.
- **Add a workspace** — click **Add Workspace**, then enter the **Workspace ID** and the **Access Token** for that specific workspace. Each workspace has its own access token.
- **Disconnect a workspace** — click the disconnect button next to any workspace to remove it.

<Tip>
  If you manage multiple Intercom workspaces (e.g., different brands or regions), you can connect each one individually. Every workspace needs its own Workspace ID and Access Token.
</Tip>

## Messaging Behaviour tab

The **Messaging Behaviour** tab has two sub-tabs:

### Channels sub-tab

Here you see every channel your agent is connected to, along with its messaging settings. For each channel you can configure:

- **Human escalation prompt** — the message shown when the conversation is handed off to a human.
- **Conversation closing message** — the message sent when the agent ends the conversation.
- **Inactivity settings** — how long the agent waits before closing an idle conversation, and the warning/closing messages.

### Files sub-tab

Configure whether the agent can receive and process files on each channel. Supported file types depend on the channel and may include images, documents, audio, and video.

<Tip>
  Use the Messaging Behaviour tab to fine-tune how your agent communicates on each channel. Different channels may need different tones, timeouts, or escalation messages.
</Tip>
