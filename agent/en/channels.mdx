## Connect your agent to real users

In the Console, **Connect** is organized in 3 sections:

- **Channels**: publish your agent in owned and external channels (Web Bubble, External Chat, WhatsApp, and others)
- **Integrations**: connect external tools/services used by your operation
- **API**: send and receive messages from your own systems

This is the step that turns your agent from "configured" into "live".

## 1) Channels

Use Channels when you want users to talk to your agent directly from chat surfaces.

### External Chat (fastest launch)

**External Chat** gives you a shareable URL you can send in seconds.

<Steps>
  <Step title="Go to Connect > Channels">
    Open your agent and navigate to the Connect page.
  </Step>
  <Step title="Open External Chat">
    Select the External Chat channel card.
  </Step>
  <Step title="Copy the public URL">
    Click the share option and copy the generated link.
  </Step>
  <Step title="Test as an end user">
    Open the link in an incognito window and run a few real use-case questions.
  </Step>
</Steps>

### Web Bubble (embed in your website)

Use **Web Bubble** when you want the agent inside your own site.

<Steps>
  <Step title="Open Bubble channel">
    In Connect > Channels, open the Bubble card.
  </Step>
  <Step title="Generate credentials/snippets">
    The Console provides the snippet and keys needed for embedding.
  </Step>
  <Step title="Add snippet to your website">
    Paste the script where your implementation team manages site code.
  </Step>
  <Step title="Customize appearance">
    Configure brand settings like launcher style and welcome behavior.
  </Step>
</Steps>

### Meta channels (WhatsApp, Messenger, Instagram)

The Connect page always displays a **Meta** section with three channel cards:

| Channel | Description |
|---|---|
| **WhatsApp** | WhatsApp Business channel via Meta |
| **Messenger** | Facebook Messenger channel via Meta |
| **Instagram** | Instagram messaging channel via Meta |

Each card shows a **Connect** button that opens the setup flow for that specific channel. You do not need to configure the Meta integration first — the cards are always visible so you can start the connection process at any time.

Once you have connected one or more accounts for a channel type, a **Channels (n)** button appears on the card showing the count of active connections. Click it to view, edit, or reconnect your existing connections.

<Steps>
  <Step title="Open the Meta section">
    Scroll to the **Meta** section on the Connect page.
  </Step>
  <Step title="Click Connect on the desired channel">
    Choose **WhatsApp**, **Messenger**, or **Instagram** and click the **+&nbsp;Connect** button.
  </Step>
  <Step title="Complete the Meta sign-up flow">
    Follow the embedded sign-up modal to authenticate with your Meta Business account and select the asset (phone number, page, or Instagram account) to connect.
  </Step>
  <Step title="Verify the connection">
    After connecting, the card updates to show the number of active channels. Click the count button to review details.
  </Step>
</Steps>

<Tip>
  For a detailed, step-by-step WhatsApp setup guide, see the dedicated [How to Integrate with WhatsApp](/integration-guides/en/how-to-integrate-to-whatsapp) integration guide.
</Tip>

## 2) Integrations

Use Integrations to connect external systems your organization depends on (for example, messaging providers, CRM/helpdesk, or custom operational services).

From the user perspective:

- You enable the integration once for your organization.
- Then you configure channels per agent.
- Optional addons can be enabled per channel when available.

This follows the backend integration model (organization integration -> agent channel -> optional addon), but the Console keeps it in a guided UI flow.

## 3) API

Use API when your own backend or app needs to send messages to the agent programmatically.

In Connect > API you can create API channels and use the API reference shown in the Console.

Typical flow:

<Steps>
  <Step title="Create an API channel">
    Go to Connect > API and create a new API channel.
  </Step>
  <Step title="Copy credentials">
    Copy the API key and endpoint provided.
  </Step>
  <Step title="Integrate in your system">
    Use the credentials and the API reference to send and receive messages.
  </Step>
</Steps>

<Note>
  Refer to the API reference in the Console for the full list of available endpoints and payload formats.
</Note>
