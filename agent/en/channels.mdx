## Connect your agent to real users

The **Connect** page is organized into two tabs:

- **Channels** — connect and manage the channels where your agent talks to users.
- **Messaging Behaviour** — configure how your agent handles messages on each connected channel.

This is the step that turns your agent from "configured" into "live".

---

## Channels tab

The Channels tab is organized in 3 sections:

- **Channels**: publish your agent in owned and external channels (Web Bubble, External Chat, WhatsApp, and others)
- **Integrations**: connect external tools/services used by your operation
- **API**: send and receive messages from your own systems

### External Chat (fastest launch)

**External Chat** gives you a shareable URL you can send in seconds.

<Steps>
  <Step title="Go to Connect > Channels">
    Open your agent and navigate to the Connect page.
  </Step>
  <Step title="Open External Chat">
    Select the External Chat channel card.
  </Step>
  <Step title="Copy the public URL">
    In the **Domain setup** tab, find the **Default URL** card and click the copy button to copy the generated link.
  </Step>
  <Step title="Test as an end user">
    Open the link in an incognito window and run a few real use-case questions.
  </Step>
</Steps>

<Tip>
  You can also connect a **custom domain** so users reach your External Chat at a branded URL like `chat.example.com/ai-agent`. See the [External Chat](/channels/en/web) page for full details on the Domain setup tab.
</Tip>

### Web Bubble (embed in your website)

Use **Web Bubble** when you want the agent inside your own site.

<Steps>
  <Step title="Open Bubble channel">
    In Connect > Channels, open the Bubble card.
  </Step>
  <Step title="Generate credentials/snippets">
    The Console provides the snippet and keys needed for embedding.
  </Step>
  <Step title="Add snippet to your website">
    Paste the script where your implementation team manages site code.
  </Step>
  <Step title="Customize appearance">
    Configure brand settings like launcher style and welcome behavior.
  </Step>
</Steps>

### Meta channels (WhatsApp, Messenger, Instagram)

The Connect page always displays a **Meta** section with three channel cards:

| Channel | Description |
|---|---|
| **WhatsApp** | WhatsApp Business channel via Meta |
| **Messenger** | Facebook Messenger channel via Meta |
| **Instagram** | Instagram messaging channel via Meta |

Each card shows a **Connect** button that opens the setup flow for that specific channel. You do not need to configure the Meta integration first — the cards are always visible so you can start the connection at any time.

<Tip>
  For step-by-step Meta integration instructions, see the [WhatsApp integration guide](/integration-guides/en/how-to-integrate-to-whatsapp).
</Tip>

## Messaging Behaviour tab

The **Messaging Behaviour** tab lets you fine-tune how your agent handles messages on each connected channel. Select a channel from the list to configure its settings.

### Debounce time

Set how long the agent waits (in seconds) before processing a new message after the last one was received. This helps group rapid consecutive messages into a single processing batch.

### File handling preferences

For each file type — **Images**, **Documents**, **Audio**, and **Video** — you can:

1. **Enable or disable** processing with a toggle switch.
2. **Define scenario cards** that tell the agent how to handle files of that type in different situations.

#### Scenario cards

When a file type is enabled, you can create one or more **scenario cards** to give the agent specific instructions for different cases. Each card has:

| Field | Description |
|---|---|
| **Title** | A short label identifying the scenario (e.g. "Invoice processing", "ID verification"). |
| **Instructions** | Detailed instructions telling the agent what to do when this scenario applies. |

This lets you handle multiple use cases per file type. For example, under **Images** you might have one scenario for "Product photo identification" and another for "Receipt OCR extraction".

<Steps>
  <Step title="Enable a file type">
    Toggle the switch next to the file type you want to configure (e.g. Images).
  </Step>
  <Step title="Add your first scenario">
    Click **Add your first scenario** in the empty-state prompt, or click the **Add scenario** card if scenarios already exist.
  </Step>
  <Step title="Fill in the card">
    Give the scenario a descriptive **title** and write the **instructions** the agent should follow.
  </Step>
  <Step title="Add more scenarios (optional)">
    Click the **+ Add scenario** card at the end of the row to create additional scenario cards. Cards scroll horizontally — use the arrow buttons to navigate when you have several.
  </Step>
  <Step title="Remove a scenario">
    Click the **trash icon** on any card to delete it.
  </Step>
  <Step title="Save">
    Click **Save** to persist your changes for the selected channel.
  </Step>
</Steps>

<Tip>
  If you previously had a single instruction for a file type, it will automatically appear as a "Default" scenario card — no data is lost during the transition.
</Tip>

<Note>
  Empty scenario cards (no title and no instructions) are automatically discarded when you save.
</Note>

---

## Key notes

- You can configure different file handling scenarios **per channel** — select the channel first, then adjust.
- Messaging Behaviour settings are saved individually per channel. Changing one channel does not affect others.
- Make sure to click **Save** after making changes to any channel's configuration.
