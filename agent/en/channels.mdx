## Connect your agent to real users

The **Connect** page is organized into two tabs:

- **Channels** — connect and manage the channels where your agent talks to users.
- **Messaging Behaviour** — configure how your agent handles conversations and messages across all channels.

This is the step that turns your agent from "configured" into "live".

---

## Channels tab

The Channels tab is organized in 3 sections:

- **Channels**: publish your agent in owned and external channels (Web Bubble, External Chat, WhatsApp, and others)
- **Integrations**: connect external tools/services used by your operation
- **API**: send and receive messages from your own systems

### External Chat (fastest launch)

**External Chat** gives you a shareable URL you can send in seconds.

<Steps>
  <Step title="Go to Connect > Channels">
    Open your agent and navigate to the Connect page.
  </Step>
  <Step title="Open External Chat">
    Select the External Chat channel card.
  </Step>
  <Step title="Copy the public URL">
    Click the share option and copy the generated link.
  </Step>
  <Step title="Test as an end user">
    Open the link in an incognito window and run a few real use-case questions.
  </Step>
</Steps>

### Web Bubble (embed in your website)

Use **Web Bubble** when you want the agent inside your own site.

<Steps>
  <Step title="Open Bubble channel">
    In Connect > Channels, open the Bubble card.
  </Step>
  <Step title="Generate credentials/snippets">
    The Console provides the snippet and keys needed for embedding.
  </Step>
  <Step title="Add snippet to your website">
    Paste the script where your implementation team manages site code.
  </Step>
  <Step title="Customize appearance">
    Configure brand settings like launcher style and welcome behavior.
  </Step>
</Steps>

### Meta channels (WhatsApp, Messenger, Instagram)

The Connect page always displays a **Meta** section with three channel cards:

| Channel | Description |
|---|---|
| **WhatsApp** | WhatsApp Business channel via Meta |
| **Messenger** | Facebook Messenger channel via Meta |
| **Instagram** | Instagram messaging channel via Meta |

Each card shows a **Connect** button that opens the setup flow for that specific channel. You do not need to configure the Meta integration first — the cards are always visible so you can start the connection process at any time.

Once you have connected one or more accounts for a channel type, a **Channels (n)** button appears on the card showing the count of active connections. Click it to view, edit, or reconnect your existing connections.

<Steps>
  <Step title="Open the Meta section">
    Scroll to the **Meta** section on the Connect page.
  </Step>
  <Step title="Click Connect on the desired channel">
    Choose **WhatsApp**, **Messenger**, or **Instagram** and follow the guided setup.
  </Step>
  <Step title="Complete authentication">
    You will be redirected to Meta to authorize the connection. Follow the on-screen prompts.
  </Step>
  <Step title="Verify the connection">
    Once connected, the channel card updates to show your active connection. Use the Playground or send a test message to confirm everything works.
  </Step>
</Steps>

<Tip>
For detailed, step-by-step instructions on connecting WhatsApp, see the [WhatsApp Integration Guide](/integration-guides/en/how-to-integrate-to-whatsapp).
</Tip>

## Messaging Behaviour tab

The **Messaging Behaviour** tab lets you control how your agent handles conversations and messages across all connected channels. It contains two sections: **Automatic Session Close** and **Channel Settings**.

### Automatic Session Close

This agent-level setting automatically closes a conversation after a period of user inactivity. It applies to **all channels** connected to your agent.

<Steps>
  <Step title="Open the Messaging Behaviour tab">
    Go to **Connect** and click the **Messaging Behaviour** tab.
  </Step>
  <Step title="Enable Automatic Session Close">
    Toggle the switch to enable or disable the feature.
  </Step>
  <Step title="Set the inactivity time">
    Use the slider or the quick-select buttons to choose how many minutes of inactivity trigger the session close (from **1 to 30 minutes**).
  </Step>
  <Step title="Review the timeline preview">
    A visual timeline shows the flow: the agent responds → the configured inactivity period passes → the session closes automatically.
  </Step>
  <Step title="Save">
    Click **Save** to apply the setting.
  </Step>
</Steps>

<Note>
When disabled, conversations remain open until they are closed manually or by another mechanism. When enabled, the farewell message (configured in **Identity**) is sent before the session closes.
</Note>

### Channel Settings

Configure **per-channel** messaging behavior. Each connected channel appears as a card showing its current settings. Click **Settings** on any channel card to open the configuration dialog.

#### Message Debounce

When a user sends multiple messages in quick succession, the agent can wait for a short period before responding—so it processes all messages together instead of replying to each one individually.

- **Time window**: Set the debounce period from **1 to 15 seconds** using the slider or quick-select buttons.
- A visual preview shows how consecutive user messages are grouped and a single response is sent after the debounce window.

<Note>
The debounce time adds to the overall response latency. For example, a 5-second debounce means the agent waits at least 5 seconds after the user's last message before responding.
</Note>

#### File Preferences

Enable and configure the types of files each channel can receive from users. Four file types are supported:

| File type | Formats |
|---|---|
| **Images** | JPG, PNG, WebP, GIF |
| **Documents** | PDF, DOCX, XLSX, CSV |
| **Audio** | Voice notes, audio files |
| **Video** | Video files |

For each file type you can:
1. **Toggle** it on or off.
2. **Add processing instructions** — a text prompt that tells the agent how to handle files of that type (e.g., "Extract text from the image and summarize it").

<Tip>
Use file processing instructions to guide the agent's behavior. For example, you could instruct it to extract data from uploaded invoices or transcribe voice notes.
</Tip>

#### Saving channel settings

After adjusting debounce time and file preferences, click **Save** in the dialog to apply the changes. The channel card updates to show badges reflecting the current configuration:
- A **debounce badge** showing the current debounce time.
- **File type badges** for each enabled file type.
