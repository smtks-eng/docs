## Connect your agent to real users

The **Connect** page is organized into two tabs:

- **Channels** — connect and manage the channels where your agent talks to users.
- **Messaging Behaviour** — configure how your agent handles messages on each connected channel.

This is the step that turns your agent from "configured" into "live".

---

## Channels tab

The Channels tab is organized in 3 sections:

- **Channels**: publish your agent in owned and external channels (Web Bubble, External Chat, WhatsApp, and others)
- **Integrations**: connect external tools/services used by your operation
- **API**: send and receive messages from your own systems

### External Chat (fastest launch)

**External Chat** gives you a shareable URL you can send in seconds.

<Steps>
  <Step title="Go to Connect > Channels">
    Open your agent and navigate to the Connect page.
  </Step>
  <Step title="Open External Chat">
    Select the External Chat channel card.
  </Step>
  <Step title="Copy the public URL">
    In the **Domain setup** tab, find the **Default URL** card and click the copy button to copy the generated link.
  </Step>
  <Step title="Test as an end user">
    Open the link in an incognito window and run a few real use-case questions.
  </Step>
</Steps>

<Tip>
  You can also connect a **custom domain** so users reach your External Chat at a branded URL like `chat.example.com/ai-agent`. See the [External Chat](/channels/en/web) page for full details on the Domain setup tab.
</Tip>

### Web Bubble (embed in your website)

Use **Web Bubble** when you want the agent inside your own site.

<Steps>
  <Step title="Open Bubble channel">
    In Connect > Channels, open the Bubble card.
  </Step>
  <Step title="Generate credentials/snippets">
    The Console provides the snippet and keys needed for embedding.
  </Step>
  <Step title="Add snippet to your website">
    Paste the script where your implementation team manages site code.
  </Step>
  <Step title="Customize appearance">
    Configure brand settings like launcher style and welcome behavior.
  </Step>
</Steps>

### Meta channels (WhatsApp, Messenger, Instagram)

The Connect page always displays a **Meta** section with three channel cards:

| Channel | Description |
|---|---|
| **WhatsApp** | WhatsApp Business channel via Meta |
| **Messenger** | Facebook Messenger channel via Meta |
| **Instagram** | Instagram messaging channel via Meta |

Each card shows a **Connect** button that opens the setup flow for that specific channel. You do not need to configure the Meta integration first — the cards are always visible so you can start the connection process directly.

## Messaging Behaviour tab

The **Messaging Behaviour** tab lets you fine-tune how your agent processes and responds to messages on each connected channel. Settings like debounce time and file-type preferences are configured **per channel**, so you can tailor behavior for WhatsApp differently than for your web widget.

### Selecting a channel

At the top of the tab you'll find a **Channel** dropdown. Open it and choose the channel you want to configure. Each option shows the channel name, type, and identifier so you can easily tell them apart.

### Copy configuration from another channel

When you have **two or more channels** connected, a second dropdown — **Copy configuration from** — appears next to the channel selector. This lets you quickly replicate another channel's messaging settings (debounce time and file preferences) into the one you're currently editing.

<Steps>
  <Step title="Select your target channel">
    Use the **Channel** dropdown to pick the channel you want to update.
  </Step>
  <Step title="Choose a source channel">
    Open the **Copy configuration from** dropdown and select the channel whose settings you want to copy.
  </Step>
  <Step title="Review the changes">
    An amber banner will appear confirming that the configuration from the source channel has been staged. The banner reads:
    *"Copying configuration from {source} → {target}. Review and Save to apply."*
    At this point the settings are loaded into the editor but **have not been saved yet**.
  </Step>
  <Step title="Save or discard">
    - Click **Save copy** to persist the copied settings.
    - Click **Discard** in the banner to revert to the channel's original configuration.
  </Step>
</Steps>

<Tip>
  The copy is not applied automatically — you always get a chance to review and adjust before saving. Switching to another channel also discards any unsaved copy.
</Tip>

### Debounce time

The **debounce time** setting controls how long the agent waits after the user's last message before it starts generating a response. This is useful on channels like WhatsApp where users often send multiple short messages in a row.

### File preferences

File preferences let you control which types of files (images, documents, audio, video) the agent can receive and process on each channel. Toggle each file type on or off depending on your use case.

<Note>
  Debounce and file-preference controls are hidden for internal channels like the Playground and API agent.
</Note>
