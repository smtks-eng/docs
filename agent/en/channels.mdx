## Connect your agent to real users

The **Connect** page is organized into two tabs:

- **Channels** — connect and manage the channels where your agent talks to users.
- **Messaging Behaviour** — configure how your agent handles messages on each connected channel.

This is the step that turns your agent from "configured" into "live".

---

## Channels tab

The Channels tab is organized in 3 sections:

- **Channels**: publish your agent in owned and external channels (Web Bubble, External Chat, WhatsApp, and others)
- **Integrations**: connect external tools/services used by your operation
- **API**: send and receive messages from your own systems

### External Chat (fastest launch)

**External Chat** gives you a shareable URL you can send in seconds.

<Steps>
  <Step title="Go to Connect > Channels">
    Open your agent and navigate to the Connect page.
  </Step>
  <Step title="Open External Chat">
    Select the External Chat channel card.
  </Step>
  <Step title="Copy the public URL">
    In the **Domain setup** tab, find the **Default URL** card and click the copy button to copy the generated link.
  </Step>
  <Step title="Test as an end user">
    Open the link in an incognito window and run a few real use-case questions.
  </Step>
</Steps>

<Tip>
  You can also connect a **custom domain** so users reach your External Chat at a branded URL like `chat.example.com/ai-agent`. See the [External Chat](/channels/en/web) page for full details on the Domain setup tab.
</Tip>

### Web Bubble (embed in your website)

Use **Web Bubble** when you want the agent inside your own site.

<Steps>
  <Step title="Open Bubble channel">
    In Connect > Channels, open the Bubble card.
  </Step>
  <Step title="Generate credentials/snippets">
    The Console provides the snippet and keys needed for embedding.
  </Step>
  <Step title="Add snippet to your website">
    Paste the script where your implementation team manages site code.
  </Step>
  <Step title="Customize appearance">
    Configure brand settings like launcher style and welcome behavior.
  </Step>
</Steps>

### Meta channels (WhatsApp, Messenger, Instagram)

The Connect page always displays a **Meta** section with three channel cards:

| Channel | Description |
|---|---|
| **WhatsApp** | WhatsApp Business channel via Meta |
| **Messenger** | Facebook Messenger channel via Meta |
| **Instagram** | Instagram messaging channel via Meta |

Each card shows a **Connect** button that opens the setup flow for that specific channel. You do not need to configure the Meta integration first — the cards are always visible so you can start the connection process at any time.

<Tip>
  For a detailed walkthrough of the WhatsApp connection process, see the [How to Integrate to WhatsApp](/integration-guides/en/how-to-integrate-to-whatsapp) guide.
</Tip>

#### WhatsApp registration status

After connecting a WhatsApp channel, a **registration status badge** appears next to the channel name. The badge tells you where the number stands in Meta's registration process:

| Status | Meaning |
|---|---|
| **Registered** | The number is fully registered and ready to send/receive messages. |
| **Not registered** | The number has not been registered with Meta yet. |
| **Verification needed** | The number must complete SMS or voice-call verification with Meta before it can be registered. |
| Other statuses | Intermediate states reported by Meta (e.g. registration in progress). |

You can click the **refresh** icon next to the badge at any time to re-check the current status from Meta.

##### Verifying a phone number

When the status is **Verification needed**, a **Verify** button appears next to the status badge. This means Meta requires you to confirm ownership of the phone number before it can be registered.

<Steps>
  <Step title="Click Verify">
    Click the **Verify** button (shield icon) next to the registration status. A popover will open.
  </Step>
  <Step title="Request a verification code">
    Choose how you want to receive the code:
    - **Send SMS** — Meta sends a 6-digit code via text message to the phone number.
    - **Voice call** — Meta delivers the code through an automated phone call.
  </Step>
  <Step title="Enter the code">
    Once you receive the 6-digit code, type it into the input field in the popover.
  </Step>
  <Step title="Submit">
    Click **Verify** to submit the code. If verification succeeds, the registration status will update automatically.
  </Step>
</Steps>

<Note>
  After a successful verification you may still need to **register** the number (using the retry/register flow with your 6-digit PIN). Verification confirms ownership; registration activates the number on the WhatsApp Business API.
</Note>

##### Registering (or re-registering) a number

When the status is anything other than **Registered** or **Verification needed**, a **Register** button appears. Clicking it opens a popover where you enter the **6-digit PIN** you want to associate with the number, then submit the registration request to Meta.

<Warning>
  Remember your PIN — you will need it for any future re-registrations of the same number.
</Warning>

## Messaging Behaviour tab

The **Messaging Behaviour** tab lets you fine-tune how your agent processes incoming messages on each channel. Settings here control things like response delay, greeting behavior, and how attachments are handled.

<Tip>
  Start with the defaults and adjust once you see real conversation patterns in the **Conversations** dashboard.
</Tip>
