Manage your AI agent's skills through two types of tools: predefined tools that come built-in with the platform, and custom tools that you can create to extend your agent's functionality.

<Tabs>
  <Tab title="Tools (Predefined)">
    The **Tools (Predefined)** tab allows you to enable or disable built-in capabilities that your AI agent can use during conversations. Each tool represents a predefined function—such as answering FAQs, sending files, or escalating to a human agent—that can be toggled on or off depending on your business needs.  

    When a tool is enabled, additional configuration may be required before the agent can use it successfully. Some tools also depend on the **type of channel** where the end user is interacting (e.g., WhatsApp, Zendesk, Freshchat).  

    ---

    ## How to Enable or Disable a Tool

    1. Navigate to **AI Agent > Design > Skills** in the left menu.  
    2. Under **Tools (Predefined)**, locate the tool you want to manage.  
    3. Use the **toggle switch** to turn the tool on or off.  
    4. If the tool requires configuration, click **Configuration**, complete the required setup, and save your changes.  
    5. Check that the tool is supported by the **channels** your agent is connected to.  

    ---

    ## Available Predefined Tools

    ### **Send Files**  
    - **Description:** Allows the agent to send files directly to the user during a conversation.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required.  

    ---

    ### **Answer FAQs**  
    - **Description:** Enables the agent to respond to frequently asked questions about your product or service.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required. You can manage and update FAQs in the **Knowledge** section of the agent.  

    ---

    ### **Collect Information**  
    - **Description:** Enables the agent to collect, verify, and store user information (e.g., name, email, phone number) for lead generation or service requests.  
    - **Toggle:** Off by default.  
    - **Configuration:** Define the fields you want the agent to collect and where the data should be stored.  

    <Note>  
    This tool is **so important** that once enabled, it activates a dedicated **Collect Information tab** in the Skills section. From there, you can create, edit, and manage information collection configurations in detail.  
    </Note>  

    <Accordion title="How to Configure Collect Information">  
    1. Go to **AI Agent > Design > Skills > Collect Information**.  
    2. Click **New Configuration** to start.  
    3. Enter a **Name** for your configuration (e.g., `registro_interesados`).  
    4. Add any **Special Instructions** to guide the agent's data-collection behavior.  
    5. Define one or more **fields** to collect (e.g., Name, Email, Phone).  
    6. Choose the **storage** or integration target for collected data.  
    7. Click **Save** to activate the configuration.  
    </Accordion>  

    ---

    ### **Escalate to Human**  
    - **Description:** Allows the agent to transfer the conversation to a live human agent when the user requests it, or when the agent cannot resolve the query.  
    - **Toggle:** Off by default.  
    - **Configuration:** Requires a compatible integration (e.g., Freshchat or Zendesk) to be connected. You can also define escalation cases—specific situations where the agent should proactively trigger escalation.  

    <Tip>
    When configuring escalation cases, you'll see an informational hint reminding you that you can also **tag FAQs to force escalation on specific topics**. Click the **Manage FAQs** link in the hint to go directly to the [Knowledge](/agent/en/knowledge) page where you can set up FAQ-based escalation triggers.
    </Tip>

    <Accordion title="How to Configure Escalation Cases">  
    1. Go to **AI Agent > Design > Skills** and enable the **Escalate to Human** tool.  
    2. Click **Configuration** to open the escalation settings.  
    3. Click **Add case** to create a new escalation case.  
    4. For each case, provide a **name** and a **situation** description that tells the agent when to escalate.  
    5. You can add multiple cases (up to the platform maximum).  
    6. Click **Save** to apply your changes.  
    </Accordion>  

    ---

    ### **Schedule Appointment**  
    - **Description:** Enables the agent to help users book or schedule appointments.  
    - **Toggle:** Off by default.  
    - **Configuration:** Requires integration with a scheduling service or calendar system.  

  </Tab>

  <Tab title="Tools (Custom)">
    The **Tools (Custom)** tab lets you create your own tools to extend your agent's functionality beyond the predefined set. Custom tools allow your agent to call external APIs, perform calculations, or trigger specific actions during conversations.

    ---

    ## How to Create a Custom Tool

    1. Navigate to **AI Agent > Design > Skills** in the left menu.  
    2. Switch to the **Tools (Custom)** tab.  
    3. Click **New Tool** to start creating.  
    4. Fill in the tool's **Name**, **Description**, and configure the **API endpoint** or action it should trigger.  
    5. Define any **parameters** the tool needs from the conversation.  
    6. Click **Save** to make the tool available.  

    <Note>  
    Custom tools require technical knowledge to set up, including familiarity with REST APIs and JSON schemas.  
    </Note>  
  </Tab>
</Tabs>
