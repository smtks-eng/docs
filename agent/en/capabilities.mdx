Manage your AI agent's skills through two types of tools: predefined tools that come built-in with the platform, and custom tools that you can create to extend your agent's functionality.

<Tabs>
  <Tab title="Tools (Predefined)">
    The **Tools (Predefined)** tab allows you to enable or disable built-in capabilities that your AI agent can use during conversations. Each tool represents a predefined function—such as answering FAQs, sending files, or escalating to a human agent—that can be toggled on or off depending on your business needs.  

    When a tool is enabled, additional configuration may be required before the agent can use it successfully. Some tools also depend on the **type of channel** where the end user is interacting (e.g., WhatsApp, Zendesk, Freshchat).  

    ---

    ## How to Enable or Disable a Tool

    1. Navigate to **AI Agent > Design > Skills** in the left menu.  
    2. Under **Tools (Predefined)**, locate the tool you want to manage.  
    3. Use the **toggle switch** to turn the tool on or off.  
    4. If the tool requires configuration, click **Configuration**, complete the required setup, and save your changes.  
    5. Check that the tool is supported by the **channels** your agent is connected to.  

    ---

    ## Available Predefined Tools

    ### **Send Files**  
    - **Description:** Allows the agent to send files directly to the user during a conversation.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required.  

    ---

    ### **Answer FAQs**  
    - **Description:** Enables the agent to respond to frequently asked questions about your product or service.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required. You can manage and update FAQs in the **Knowledge** section of the agent.  

    ---

    ### **Collect Information**  
    - **Description:** Enables the agent to collect, verify, and store user information (e.g., name, email, phone number) for lead generation or service requests.  
    - **Toggle:** Off by default.  
    - **Configuration:** Define the fields you want the agent to collect and where the data should be stored.  

    <Note>  
    This tool is **so important** that once enabled, it activates a dedicated **Collect Information tab** in the Skills section. From there, you can create, edit, and manage information collection configurations in detail.  
    </Note>  

    <Accordion title="How to Configure Collect Information">  
    1. Go to **AI Agent > Design > Skills > Collect Information**.  
    2. Click **New Configuration** to start.  
    3. Enter a **Name** for your configuration (e.g., `registro_interesados`).  
    4. Add any **Special Instructions** to guide the agent's behavior.  
    5. Add your fields — including the **type** (text, number, date, boolean) and instructions.  
    6. Click **Save** to activate the configuration.  
    </Accordion>  

    ---

    ### **Human Escalation**  
    - **Description:** Transfers the conversation to a live human agent when needed.  
    - **Toggle:** Off by default.  
    - **Configuration:** Requires an active **Freshchat** or **Zendesk** integration.  

    <Warning>  
    This tool only works with supported integrations. If no compatible integration is connected, it cannot be enabled.  
    </Warning>  

  </Tab>

  <Tab title="Custom Tools">
    The **Custom Tools** tab allows you to create your own tools (skills) that extend what your AI agent can do. Custom tools let you connect your agent to external APIs so it can perform actions like looking up order statuses, querying databases, booking appointments, or any other HTTP-based operation.

    ---

    ## Creating a Custom Tool

    To create a new custom tool:

    1. Go to **AI Agent > Design > Skills > Custom Tools**.  
    2. Click **Create Tool**.  
    3. Fill in the tool creation form (see below).  
    4. Click **Save** to register the tool.  

    ### Tool Creation Form Fields

    | Field | Description |
    |---|---|
    | **Name** | A descriptive name for the tool. |
    | **Description** | What this tool does — the agent uses this to decide when to invoke it. |
    | **HTTP Method** | The HTTP verb: GET, POST, PUT, DELETE, or PATCH. |
    | **URL** | The endpoint the tool will call. |
    | **Output Format** | Expected response format: JSON, Text, XML, CSV, or HTML. |
    | **Properties** | Input parameters for the API call (see below). |

    ### Tool Properties

    Each property defines an input parameter for your external API. When adding a property, you configure:

    | Setting | Description |
    |---|---|
    | **Key** | The parameter name (e.g., `order_id`, `api_key`). |
    | **Description** | Explains what this parameter is for. |
    | **Value Type** | The data type: Text, Number, Date, Secret, or Object. |
    | **Location** | Where the parameter is sent: **Header**, **Query**, **Cookie**, **Body**, **Path**, or **Auth Bearer**. |
    | **Value At** | When/how the value is provided (see table below). |
    | **Quantity** | Whether the parameter accepts a **Single** value or **Multiple** values. |
    | **Required** | Whether this property must have a value for the tool to work. |

    #### Value At Options

    The **Value At** setting determines how and when the property value is provided:

    | Option | Description |
    |---|---|
    | **Configuration** | The value is set once during tool setup. Ideal for API keys, static IDs, or secrets that don't change per conversation. |
    | **Conversation** | The value is provided dynamically each time the tool is invoked — the agent extracts it from the conversation context. |
    | **End User** | The value is resolved at runtime from the end user associated with the channel. This is useful for passing user-specific identifiers (like an external user ID or metadata) without requiring the user to provide them in the conversation. |

    <Tip>
    Use **Configuration** for secrets and static values, **Conversation** for dynamic data the user provides, and **End User** for user-specific identifiers that should be resolved automatically from the channel.
    </Tip>

    ---

    ## Configuring a Custom Tool

    After a custom tool is created and assigned to your agent, click **Configure** to set it up. The configuration dialog has two sections:

    ### Configuration Values

    For every property that has **Value At = Configuration**, you'll see an input field where you enter the static value (e.g., an API key or a base URL). Properties marked as required must be filled in for the tool to become active.

    ### Channel Assignment

    Select which channels should have access to this tool. You can enable or disable the tool per channel.

    #### Per-Channel End User Values

    If your tool has properties with **Value At = End User**, each selected channel will display an additional **End User Values** section. This lets you configure how each end-user property is resolved on a per-channel basis.

    For each end-user property, you choose a **source**:

    | Source | Description |
    |---|---|
    | **External User ID** | Uses the end user's external identifier as the value. This is the default. |
    | **Metadata Key** | Reads the value from a specific key in the end user's metadata. When selected, an additional text field appears where you enter the metadata key name (e.g., `phone`, `account_id`, `plan`). |

    <Note>
    Each channel can map the same end-user property to a different source. For example, a WhatsApp channel might resolve a `customer_id` property from the External User ID, while a web channel might read it from a metadata key called `crm_id`.
    </Note>

    <Warning>
    If a required end-user property is not configured for a channel, the tool status will show as **missing_config** and will not be active until all required properties are configured across all selected channels.
    </Warning>

    ---

    ## Tool Status

    Each custom tool has a status indicator:

    | Status | Meaning |
    |---|---|
    | **Active** | All required properties have values and the tool is ready to use. |
    | **Missing Config** | One or more required properties (configuration or end-user) are missing values. |
    | **Review** | The tool needs review before it can be activated. |
