Manage your AI agent's skills through two types of tools: predefined tools that come built-in with the platform, and custom tools that you can create to extend your agent's functionality.

<Tabs>
  <Tab title="Tools (Predefined)">
    The **Tools (Predefined)** tab allows you to enable or disable built-in capabilities that your AI agent can use during conversations. Each tool represents a predefined function—such as answering FAQs, sending files, or escalating to a human agent—that can be toggled on or off depending on your business needs.  

    When a tool is enabled, additional configuration may be required before the agent can use it successfully. Some tools also depend on the **type of channel** where the end user is interacting (e.g., WhatsApp, Zendesk, Freshchat).  

    ---

    ## How to Enable or Disable a Tool

    1. Navigate to **AI Agent > Design > Skills** in the left menu.  
    2. Under **Tools (Predefined)**, locate the tool you want to manage.  
    3. Use the **toggle switch** to turn the tool on or off.  
    4. If the tool requires configuration, click **Configuration**, complete the required setup, and save your changes.  
    5. Check that the tool is supported by the **channels** your agent is connected to.  

    ---

    ## Available Predefined Tools

    ### **Send Files**  
    - **Description:** Allows the agent to send files directly to the user during a conversation.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required.  

    ---

    ### **Answer FAQs**  
    - **Description:** Enables the agent to respond to frequently asked questions about your product or service.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required. You can manage and update FAQs in the **Knowledge** section of the agent.  

    ---

    ### **Collect Information**  
    - **Description:** Enables the agent to collect, verify, and store user information (e.g., name, email, phone number) for lead generation or service requests.  
    - **Toggle:** Off by default.  
    - **Configuration:** Define the fields you want the agent to collect and where the data should be stored.  

    <Note>  
    This tool is **so important** that once enabled, it activates a dedicated **Collect Information tab** in the Skills section. From there, you can create, edit, and manage information collection configurations in detail.  
    </Note>  

    <Accordion title="How to Configure Collect Information">  
    1. Go to **AI Agent > Design > Skills > Collect Information**.  
    2. Click **New Configuration** to start.  
    3. Enter a **Name** for your configuration (e.g., `registro_interesados`).  
    4. Add any **Special Instructions** to guide the agent's behavior during the collection.  
    5. Click on **Add Field** and define the information you want to collect.  
    6. Set each field's **type** (text, phone, email, number, date, etc.).  
    7. Save your configuration.  
    8. Assign the configuration to a **channel** so the agent knows where to apply it.  
    </Accordion>  

    ---

    ### **Human Escalation**  
    - **Description:** Allows the agent to escalate the conversation to a human agent.  
    - **Toggle:** Available once Freshchat or Zendesk is connected.  
    - **Configuration:** You must connect and configure **Freshchat** or **Zendesk** integration first. Once enabled, you can set a custom escalation message.  

    <Note>  
    This tool only appears when a compatible integration (Freshchat or Zendesk) has been connected.  
    </Note>  

    ---

    ### **Schedule Messages**  
    - **Description:** Allows the agent to schedule messages to be sent to users at a later time.  
    - **Toggle:** Off by default.  
    - **Configuration:** No setup required.  

    ---

    ### **Send Emails**  
    - **Description:** Allows the agent to send emails on behalf of the business.  
    - **Toggle:** Off by default.  
    - **Configuration:** Requires email provider configuration.  

  </Tab>

  <Tab title="Custom Tools">

    The **Custom Tools** tab lets you extend your AI agent's abilities by connecting it to external APIs. Custom tools allow the agent to perform actions such as looking up order statuses, creating support tickets, fetching account details, or any other operation exposed by an API.

    The Custom Tools section is organized in three sub-tabs:

    - **Custom** — create and manage your custom tool definitions.
    - **Assigned** — assign tools to your agent and configure them per channel.
    - **Invocations** — view a log of every time a tool was called, with request/response details.

    ---

    ## Creating a Custom Tool with the Skill Builder

    When you click **Create Tool** in the Custom sub-tab, you are taken to the **Skill Builder** — a dedicated full-page, guided experience for defining your tool step by step.

    <Note>
    The Skill Builder replaces the previous modal-based form. You now have a full page with more space, live previews, and a built-in test panel.
    </Note>

    ### Getting to the Skill Builder

    <Steps>
      <Step title="Navigate to Custom Tools">
        Go to **AI Agent > Design > Skills** and select the **Custom Tools** tab, then the **Custom** sub-tab.
      </Step>
      <Step title="Click Create Tool">
        Click the **Create Tool** button. You will be taken to the Skill Builder page.
      </Step>
    </Steps>

    To **edit** an existing tool, click the edit button on its card in the Custom sub-tab. You will be taken to the same Skill Builder page pre-filled with the tool's current configuration.

    ---

    ### Skill Builder Overview

    The Skill Builder walks you through configuring your custom tool with the following sections:

    #### 1. Name & Description
    Give your tool a **title** (a human-readable display name) and a **description** that tells the agent when and why to use this tool. The description is critical — it helps the AI decide when to invoke the tool during a conversation.

    #### 2. HTTP Method & URL
    Select the HTTP method (**GET**, **POST**, **PUT**, **PATCH**, or **DELETE**) and enter the endpoint URL your tool will call. If your URL contains path parameters, use curly-brace placeholders (e.g., `https://api.example.com/orders/{order_id}`).

    #### 3. Inputs (Parameters)
    Define the data your tool needs to make the API call. For each input you add, configure:

    | Setting | Description |
    |---|---|
    | **Name** | The parameter name (used in the API request) |
    | **Source** | Where the value comes from (see table below) |
    | **Type** | Data type — text, whole number, decimal, boolean, list, or nested object |
    | **Required** | Whether the parameter is mandatory |
    | **Description** | Helps the agent understand what this parameter is for |
    | **Location** | Where the parameter is placed in the request — query string, path, body, or header. This is auto-inferred but can be set manually. |

    **Available input sources:**

    | Source | Tag | Description |
    |---|---|---|
    | The user says it in the chat | `conversation` | The agent extracts the value from the conversation automatically. |
    | It's who we're chatting with | `context` | Contextual info about the current user, set when the agent starts (e.g., user ID, session info). |
    | Always the same value | `fixed` | A constant value you set once; sent identically on every call. The agent and customer never see it. |
    | A secret kept safe | `env` | A sensitive value (like an API key) stored as an environment variable. Configured once and kept secure. |

    <Tip>
    For parameters that come from the conversation, write a clear **description** so the agent knows what to ask the user for (e.g., "The customer's order number, usually a 6-digit code").
    </Tip>

    <Note>
    The Skill Builder also shows additional source options for advanced use cases (state, store, config). These are mapped to the closest supported backend value and surfaced with a warning if the backend doesn't fully support them yet.
    </Note>

    #### 4. Output
    Configure what the tool returns:
    - **Return type** — choose whether the API response is a bundle of details (dict), a list, a number, or plain text.
    - **Prompt** — instructions for the agent on how to interpret and present the response to the user.
    - **Example response** — an optional sample JSON response so the agent understands the response structure.

    #### 5. Error Handling
    Provide instructions for the agent on what to do when the API call fails (e.g., apologize and offer to retry, escalate to a human agent, etc.).

    #### 6. Timeout
    Set the maximum number of seconds to wait for the API to respond (default: 30 seconds).

    ---

    ### Live Previews

    As you fill in the Skill Builder, two live previews update in real time:

    - **Python preview** — shows the generated Python/Pydantic code that represents your tool definition. This is useful for developers who want to understand or validate the underlying tool structure.
    - **cURL preview** — shows the equivalent cURL command for the API call, including all configured parameters, headers, and body data.

    You can copy either preview to your clipboard with one click.

    ---

    ### Testing Your Tool

    The Skill Builder includes a built-in **Test** panel. After configuring your tool, you can send a test request directly from the builder to verify that the API responds correctly before saving.

    ---

    ### Saving

    When you're done, click **Save** (or **Update** if editing). You'll be redirected back to the Custom Tools list. From there, you can assign the tool to your agent.

    ---

    ## Assigning a Tool to Your Agent

    After creating a custom tool, you need to assign it to your agent before the agent can use it.

    <Steps>
      <Step title="Go to the Assigned sub-tab">
        In **Skills > Custom Tools**, switch to the **Assigned** sub-tab.
      </Step>
      <Step title="Click Assign Tool">
        Click the **Assign Tool** button.
      </Step>
      <Step title="Select the tool and agent">
        Choose the tool you want to assign and confirm.
      </Step>
    </Steps>

    Once assigned, you can configure the tool per channel and set any required environment variables.

    ---

    ## Tool Invocations

    The **Invocations** sub-tab shows a log of every time any custom tool was called by the agent, including:
    - Tool name
    - Execution status (success, error, timeout)
    - Timestamp
    - Full request and response data (expandable)

    <Tip>
    Use the Invocations log to debug tool behavior and verify that parameters are being sent correctly. You can also see invocations in the global **Task Log** under Supervise.
    </Tip>
