Manage your AI agent's skills through two types of tools: predefined tools that come built-in with the platform, and custom tools that you can create to extend your agent's functionality.

<Tabs>
  <Tab title="Tools (Predefined)">
    The **Tools (Predefined)** tab allows you to enable or disable built-in capabilities that your AI agent can use during conversations. Each tool represents a predefined function—such as answering FAQs, sending files, or escalating to a human agent—that can be toggled on or off depending on your business needs.  

    When a tool is enabled, additional configuration may be required before the agent can use it successfully. Some tools also depend on the **type of channel** where the end user is interacting (e.g., WhatsApp, Zendesk, Freshchat).  

    ---

    ## How to Enable or Disable a Tool

    1. Navigate to **AI Agent > Design > Skills** in the left menu.  
    2. Under **Tools (Predefined)**, locate the tool you want to manage.  
    3. Use the **toggle switch** to turn the tool on or off.  
    4. If the tool requires configuration, click **Configuration**, complete the required setup, and save your changes.  
    5. Check that the tool is supported by the **channels** your agent is connected to.  

    ---

    ## Available Predefined Tools

    ### **Send Files**  
    - **Description:** Allows the agent to send files directly to the user during a conversation.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required.  

    ---

    ### **Answer FAQs**  
    - **Description:** Enables the agent to respond to frequently asked questions about your product or service.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required. You can manage and update FAQs in the **Knowledge** section of the agent.  

    ---

    ### **Collect Information**  
    - **Description:** Enables the agent to collect, verify, and store user information (e.g., name, email, phone number) for lead generation or service requests.  
    - **Toggle:** Off by default.  
    - **Configuration:** Define the fields you want the agent to collect and where the data should be stored.  

    <Note>  
    This tool is **so important** that once enabled, it activates a dedicated **Collect Information tab** in the Skills section. From there, you can create, edit, and manage information collection configurations in detail.  
    </Note>  

    <Accordion title="How to Configure Collect Information">  
    1. Go to **AI Agent > Design > Skills > Collect Information**.  
    2. Click **New Configuration** to start.  
    3. Enter a **Name** for your configuration (e.g., `registro_interesados`).  
    4. Add any **Special Instructions** to guide the agent's behavior during collection.  
    5. Define the **Fields** you want to collect, including the field name, type, and prompt instructions.  
    6. Assign the configuration to one or more **Channels**.  
    7. Click **Save** to activate.  
    </Accordion>  

    ---

    ### **Get Contact from HubSpot**
    - **Description:** Allows the agent to look up a contact in your connected HubSpot CRM during a conversation. The agent can retrieve contact details (such as name, email, company, lifecycle stage, etc.) and use them to personalize responses or route conversations.
    - **Toggle:** Off by default.
    - **Configuration:** Specify which HubSpot contact properties the agent should retrieve.

    <Note>
    This tool requires an active **HubSpot integration** configured in **AI Agent > Connect > Integrations**. Make sure your HubSpot account is connected before enabling this skill.
    </Note>

    <Accordion title="How to Configure Get Contact from HubSpot">
    1. Go to **AI Agent > Design > Skills** and enable the **Get Contact from HubSpot** tool.
    2. Click **Configuration** to open the properties editor.
    3. You will see an inline list editor where you can define which HubSpot contact properties the agent should retrieve.
    4. Click the **+** button to add a new property.
    5. Type the **HubSpot property identifier** (the internal name used by HubSpot, e.g. `firstname`, `email`, `company`, `lifecyclestage`).
    6. Use the **pencil icon** to edit an existing property, or the **trash icon** to remove one.
    7. Press **Enter** or click the **checkmark icon** to confirm your edit.
    8. Click **Save** when you're done.

    If you leave the properties list empty, the agent will use a default set of properties.
    </Accordion>

    <Tip>
    You can find a list of available HubSpot contact properties in your HubSpot account under **Settings > Properties > Contact properties**. Use the **internal name** (not the label) when adding properties here.
    </Tip>

    ---

    ### **Human Escalation**  
    - **Description:** Enables the agent to escalate a conversation to a live human agent when it detects the need.  
    - **Toggle:** Off by default.  
    - **Configuration:** Requires a compatible integration (Freshchat, Zendesk, or the built-in Live Chat). Define when and how the agent should escalate.  

    ---

  </Tab>

  <Tab title="Custom Tools">
    **Custom Tools** allow you to extend your agent's capabilities beyond the built-in predefined tools. You can create tools that call external APIs, perform calculations, or trigger actions in third-party systems.

    Custom tools are defined using a JSON schema that describes the tool's inputs, outputs, and execution logic.

    ---

    ## Creating a Custom Tool

    1. Navigate to **AI Agent > Design > Skills**.  
    2. Switch to the **Custom Tools** tab.  
    3. Click **New Tool** to start building.  
    4. Define the tool's **name**, **description**, and **parameters**.  
    5. Configure the **API endpoint** or execution logic.  
    6. Save and test using the **Playground**.  

  </Tab>
</Tabs>
