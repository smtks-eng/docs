Manage your AI agent's skills through two types of tools: predefined tools that come built-in with the platform, and custom tools that you can create to extend your agent's functionality.

<Tabs>
  <Tab title="Tools (Predefined)">
    The **Tools (Predefined)** tab allows you to enable or disable built-in capabilities that your AI agent can use during conversations. Each tool represents a predefined function—such as answering FAQs, sending files, or escalating to a human agent—that can be toggled on or off depending on your business needs.  

    When a tool is enabled, additional configuration may be required before the agent can use it successfully. Some tools also depend on the **type of channel** where the end user is interacting (e.g., WhatsApp, Zendesk, Freshchat).  

    ---

    ## How to Enable or Disable a Tool

    1. Navigate to **AI Agent > Design > Skills** in the left menu.  
    2. Under **Tools (Predefined)**, locate the tool you want to manage.  
    3. Use the **toggle switch** to turn the tool on or off.  
    4. If the tool requires configuration, click **Configuration**, complete the required setup, and save your changes.  
    5. Check that the tool is supported by the **channels** your agent is connected to.  

    ---

    ## Available Predefined Tools

    ### **Send Files**  
    - **Description:** Allows the agent to send files directly to the user during a conversation.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required.  

    ---

    ### **Answer FAQs**  
    - **Description:** Enables the agent to respond to frequently asked questions about your product or service.  
    - **Default:** Always enabled.  
    - **Configuration:** No setup required. You can manage and update FAQs in the **Knowledge** section of the agent.  

    ---

    ### **Collect Information**  
    - **Description:** Enables the agent to collect, verify, and store user information (e.g., name, email, phone number) for lead generation or service requests.  
    - **Toggle:** Off by default.  
    - **Configuration:** Define the fields you want the agent to collect and where the data should be stored.  

    <Note>  
    This tool is **so important** that once enabled, it activates a dedicated **Collect Information tab** in the Skills section. From there, you can create, edit, and manage information collection configurations in detail.  
    </Note>  

    <Accordion title="How to Configure Collect Information">  
    1. Go to **AI Agent > Design > Skills > Collect Information**.  
    2. Click **New Configuration** to start.  
    3. Enter a **Name** for your configuration (e.g., `registro_interesados`).  
    4. Add any **Special Instructions** to guide the agent during the collection process.  
    5. Define the **fields** you want to collect from users.  
    6. Click **Save** when you're done.  
    </Accordion>  

    ---

    ### **Human Escalation**  
    - **Description:** Allows the agent to transfer the conversation to a human agent when the user requests it or when the agent cannot resolve the query.  
    - **Toggle:** Off by default.  
    - **Configuration:** Connect a compatible integration (Freshchat or Zendesk) and configure escalation settings such as availability schedules.  

    ---

    ### **Retrieve Knowledge**  
    - **Description:** Enables the agent to search your uploaded knowledge base during a conversation.  
    - **Toggle:** Off by default.  
    - **Configuration:** Make sure your agent has knowledge uploaded in the **Knowledge** section. No additional setup required beyond having content available.

  </Tab>

  <Tab title="Custom Tools">
    The **Custom Tools** tab lets you create your own tools to extend your agent's capabilities beyond the predefined set. Custom tools allow your agent to call external APIs, retrieve data from your systems, and perform actions tailored to your business logic.

    ---

    ## Creating a Custom Tool

    1. Navigate to **AI Agent > Design > Skills**.  
    2. Switch to the **Custom Tools** tab.  
    3. Click **New Tool** to start creating a custom tool.  
    4. Fill in the tool name, description, and configuration.  
    5. Save your tool.

    ---

    ## Custom Tool Configuration

    When you select a custom tool, a detail panel opens with two tabs: **Configuration** and **Channels**.

    ### Configuration Tab

    The Configuration tab contains all the settings that control how your tool works. Depending on how the tool was set up, you may see a combination of standard input fields (text, secret, URL, dropdown, etc.) and advanced component editors described below.

    #### Input Field Types

    Configuration forms support several field types:

    | Type | Description |
    |---|---|
    | **Text** | A single-line text input for values like URLs or names. |
    | **Number** | A numeric input field. Values are stored as real numbers so backend consumers receive the correct data type. |
    | **Secret** | A masked input for sensitive values like API keys. Secrets are encrypted before being stored. |
    | **Textarea** | A multi-line text area for longer content like instructions or templates. |
    | **Options** | A dropdown selector for choosing from predefined values (e.g., timezone). |

    ---

    ### Property Editor

    Tools that call external HTTP endpoints (such as the **Get User Context** tool) include a **Property Editor** component inside the Configuration tab. The property editor lets you define every parameter sent in the HTTP request and control where each value comes from at runtime.

    #### HTTP Method

    At the top of the property editor, select the **HTTP Method** used by the endpoint:

    - `GET`
    - `POST`
    - `PUT`
    - `DELETE`
    - `PATCH`

    #### Adding and Managing Properties

    Click **Add Property** to add a new request parameter. Each property has the following fields:

    | Field | Description |
    |---|---|
    | **Key** | The parameter name (e.g., `user_id`, `X-Api-Key`). The placeholder changes depending on the property type to guide you. |
    | **Description** | A short note for your own reference describing what this property does. |
    | **Type** | Where the parameter is sent: **Header**, **Query**, **Cookie**, **Body**, **Path**, or **Auth Bearer**. Path parameters replace `{key}` placeholders in the URL. |
    | **Value Type** | The data format: **Text**, **Number**, **Secret**, **Date**, or **Object**. |
    | **Value At** | Where the value comes from at runtime (see table below). |
    | **Quantity** | Whether the parameter accepts a **Single** value or **Multiple** values. |
    | **Required** | Check this if the parameter is mandatory for the request to succeed. |

    #### Value At Options

    The **Value At** dropdown determines how the property's value is resolved when the tool runs:

    | Option | Description |
    |---|---|
    | **Configuration** | The value is set here in the tool configuration and stays fixed. You will be prompted to enter the value in the **Configuration Values** section below the property list. |
    | **External User ID** | The value is automatically filled with the end user's external ID at runtime. |
    | **Metadata Key** | The value is read from the end user's metadata. When selected, an additional field appears where you enter the metadata key (e.g., `customer_id`). |
    | **Conversation** | The agent extracts the value from the ongoing conversation context. |

    <Tip>
    Properties marked as **Configuration** with a **Secret** value type will have their values encrypted before saving—ideal for API keys and tokens.
    </Tip>

    #### Configuration Values Section

    When one or more properties have **Value At** set to **Configuration**, a dedicated section appears below the property list. Here you enter the fixed value for each of those properties. Secret-type properties display a masked input for security.

    #### Removing a Property

    Click the **minus icon** (−) on the top-right corner of any property card to remove it.

    ---

    ### Error Handling

    Some tools include an **Error Handling** section in the Configuration tab. Use the text area to describe how the agent should behave if the endpoint call fails—for example, showing a fallback message to the user or retrying the request.

    ---

    ### Channels Tab

    The **Channels** tab lets you control which connected channels can use this tool. Enable or disable the tool for each channel independently.

    <Note>
    All configuration components (property editor, error handling, and standard inputs) share the same **Save** button. Make sure to click **Save** after making any changes.
    </Note>

  </Tab>
