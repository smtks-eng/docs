Administra las habilidades de tu agente de IA a través de dos tipos de herramientas: herramientas predefinidas que vienen integradas con la plataforma y herramientas personalizadas que puedes crear para extender la funcionalidad de tu agente.

<Tabs>
  <Tab title="Herramientas (Predefinidas)">
    La pestaña **Herramientas (Predefinidas)** te permite habilitar o deshabilitar capacidades integradas que tu agente de IA puede usar durante las conversaciones. Cada herramienta representa una función predefinida—como responder preguntas frecuentes, enviar archivos o escalar a un agente humano—que puede activarse o desactivarse según las necesidades de tu negocio.

    Cuando una herramienta está habilitada, puede requerir configuración adicional antes de que el agente pueda usarla exitosamente. Algunas herramientas también dependen del **tipo de canal** donde el usuario final está interactuando (por ejemplo, WhatsApp, Zendesk, Freshchat).

    ---

    ## Cómo Habilitar o Deshabilitar una Herramienta

    1. Navega a **Agente IA > Diseño > Habilidades** en el menú lateral.
    2. En **Herramientas (Predefinidas)**, localiza la herramienta que deseas gestionar.
    3. Usa el **interruptor** para activar o desactivar la herramienta.
    4. Si la herramienta requiere configuración, haz clic en **Configuración**, completa la configuración requerida y guarda los cambios.
    5. Verifica que la herramienta sea compatible con los **canales** a los que tu agente está conectado.

    ---

    ## Herramientas Predefinidas Disponibles

    ### **Enviar Archivos**
    - **Descripción:** Permite al agente enviar archivos directamente al usuario durante una conversación.
    - **Predeterminado:** Siempre habilitado.
    - **Configuración:** No requiere configuración.

    ---

    ### **Responder Preguntas Frecuentes**
    - **Descripción:** Permite al agente responder preguntas frecuentes sobre tu producto o servicio.
    - **Predeterminado:** Siempre habilitado.
    - **Configuración:** No requiere configuración. Puedes gestionar y actualizar las preguntas frecuentes en la sección **Conocimiento** del agente.

    ---

    ### **Recopilar Información**
    - **Descripción:** Permite al agente recopilar, verificar y almacenar información del usuario (por ejemplo, nombre, correo electrónico, número de teléfono) para generación de leads o solicitudes de servicio.
    - **Interruptor:** Desactivado por defecto.
    - **Configuración:** Define los campos que deseas que el agente recopile y dónde se debe almacenar la información.

    <Note>
    Esta herramienta es **tan importante** que una vez habilitada, activa una **pestaña dedicada de Recopilar información** en la sección de Habilidades. Desde allí, puedes crear, editar y gestionar configuraciones de recopilación de información en detalle.
    </Note>

    <Accordion title="Cómo Configurar Recopilar Información">
    1. Ve a **Agente IA > Diseño > Habilidades > Recopilar Información**.
    2. Haz clic en **Nueva Configuración** para comenzar.
    3. Ingresa un **Nombre** para tu configuración (ej. `registro_interesados`).
    4. Agrega **Instrucciones Especiales** para guiar el comportamiento del agente.
    5. Agrega tus campos — incluyendo el **tipo** (texto, número, fecha, booleano) e instrucciones.
    6. Haz clic en **Guardar** para activar la configuración.
    </Accordion>

    ---

    ### **Escalación Humana**
    - **Descripción:** Transfiere la conversación a un agente humano cuando sea necesario.
    - **Interruptor:** Desactivado por defecto.
    - **Configuración:** Requiere una integración activa de **Freshchat** o **Zendesk**.

    <Warning>
    Esta herramienta solo funciona con integraciones compatibles. Si no hay una integración compatible conectada, no se puede habilitar.
    </Warning>

  </Tab>

  <Tab title="Herramientas Personalizadas">
    La pestaña **Herramientas Personalizadas** te permite crear tus propias herramientas (habilidades) que extienden lo que tu agente de IA puede hacer. Las herramientas personalizadas te permiten conectar tu agente a APIs externas para que pueda realizar acciones como consultar estados de pedidos, consultar bases de datos, agendar citas o cualquier otra operación basada en HTTP.

    ---

    ## Creando una Herramienta Personalizada

    Para crear una nueva herramienta personalizada:

    1. Ve a **Agente IA > Diseño > Habilidades > Herramientas Personalizadas**.
    2. Haz clic en **Crear Herramienta**.
    3. Completa el formulario de creación de herramientas (ver abajo).
    4. Haz clic en **Guardar** para registrar la herramienta.

    ### Campos del Formulario de Creación

    | Campo | Descripción |
    |---|---|
    | **Nombre** | Un nombre descriptivo para la herramienta. |
    | **Descripción** | Qué hace esta herramienta — el agente lo usa para decidir cuándo invocarla. |
    | **Método HTTP** | El verbo HTTP: GET, POST, PUT, DELETE o PATCH. |
    | **URL** | El endpoint que la herramienta llamará. |
    | **Formato de Salida** | Formato de respuesta esperado: JSON, Text, XML, CSV o HTML. |
    | **Propiedades** | Parámetros de entrada para la llamada API (ver abajo). |

    ### Propiedades de la Herramienta

    Cada propiedad define un parámetro de entrada para tu API externa. Al agregar una propiedad, configuras:

    | Configuración | Descripción |
    |---|---|
    | **Clave** | El nombre del parámetro (ej. `order_id`, `api_key`). |
    | **Descripción** | Explica para qué es este parámetro. |
    | **Tipo de Valor** | El tipo de dato: Texto, Número, Fecha, Secreto u Objeto. |
    | **Ubicación** | Dónde se envía el parámetro: **Header**, **Query**, **Cookie**, **Body**, **Path** o **Auth Bearer**. |
    | **Valor En** | Cuándo/cómo se proporciona el valor (ver tabla abajo). |
    | **Cantidad** | Si el parámetro acepta un valor **Único** o **Múltiples** valores. |
    | **Requerido** | Si esta propiedad debe tener un valor para que la herramienta funcione. |

    #### Opciones de Valor En

    La configuración **Valor En** determina cómo y cuándo se proporciona el valor de la propiedad:

    | Opción | Descripción |
    |---|---|
    | **Configuración** | El valor se establece una vez durante la configuración de la herramienta. Ideal para API keys, IDs estáticos o secretos que no cambian por conversación. |
    | **Conversación** | El valor se proporciona dinámicamente cada vez que se invoca la herramienta — el agente lo extrae del contexto de la conversación. |
    | **Usuario Final** | El valor se resuelve en tiempo de ejecución a partir del usuario final asociado al canal. Esto es útil para pasar identificadores específicos del usuario (como un ID de usuario externo o metadatos) sin requerir que el usuario los proporcione en la conversación. |

    <Tip>
    Usa **Configuración** para secretos y valores estáticos, **Conversación** para datos dinámicos que el usuario proporciona, y **Usuario Final** para identificadores específicos del usuario que deben resolverse automáticamente desde el canal.
    </Tip>

    ---

    ## Configurando una Herramienta Personalizada

    Después de crear una herramienta personalizada y asignarla a tu agente, haz clic en **Configurar** para configurarla. El diálogo de configuración tiene dos secciones:

    ### Valores de Configuración

    Para cada propiedad que tenga **Valor En = Configuración**, verás un campo de entrada donde ingresas el valor estático (ej. una API key o una URL base). Las propiedades marcadas como requeridas deben completarse para que la herramienta se active.

    ### Asignación de Canales

    Selecciona qué canales deben tener acceso a esta herramienta. Puedes habilitar o deshabilitar la herramienta por canal.

    #### Valores de Usuario Final por Canal

    Si tu herramienta tiene propiedades con **Valor En = Usuario Final**, cada canal seleccionado mostrará una sección adicional de **Valores del Usuario Final**. Esto te permite configurar cómo se resuelve cada propiedad de usuario final en cada canal de forma independiente.

    Para cada propiedad de usuario final, eliges un **origen**:

    | Origen | Descripción |
    |---|---|
    | **ID de Usuario Externo** | Usa el identificador externo del usuario final como valor. Este es el valor predeterminado. |
    | **Clave de Metadatos** | Lee el valor de una clave específica en los metadatos del usuario final. Al seleccionarlo, aparece un campo de texto adicional donde ingresas el nombre de la clave de metadatos (ej. `phone`, `account_id`, `plan`). |

    <Note>
    Cada canal puede mapear la misma propiedad de usuario final a un origen diferente. Por ejemplo, un canal de WhatsApp podría resolver una propiedad `customer_id` desde el ID de Usuario Externo, mientras que un canal web podría leerla de una clave de metadatos llamada `crm_id`.
    </Note>

    <Warning>
    Si una propiedad de usuario final requerida no está configurada para un canal, el estado de la herramienta se mostrará como **missing_config** y no estará activa hasta que todas las propiedades requeridas estén configuradas en todos los canales seleccionados.
    </Warning>

    ---

    ## Estado de la Herramienta

    Cada herramienta personalizada tiene un indicador de estado:

    | Estado | Significado |
    |---|---|
    | **Activa** | Todas las propiedades requeridas tienen valores y la herramienta está lista para usar. |
    | **Configuración Faltante** | Una o más propiedades requeridas (de configuración o de usuario final) no tienen valores. |
    | **Revisión** | La herramienta necesita revisión antes de poder activarse. |
