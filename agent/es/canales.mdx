## Conecta tu agente con usuarios reales

La página **Conecta** se organiza en dos pestañas:

- **Canales** — conecta y administra los canales donde tu agente habla con los usuarios.
- **Comportamiento de Mensajería** — configura cómo tu agente maneja los mensajes en cada canal conectado.

Este paso convierte tu agente de «configurado» a «en producción».

---

## Pestaña de Canales

La pestaña de Canales se organiza en tres bloques:

- **Canales** — publica tu agente en canales propios y externos (burbuja web, chat externo, WhatsApp y otros; en la interfaz algunos nombres pueden aparecer en inglés).
- **Integraciones** — conecta herramientas o servicios externos de tu operación.
- **API** — envía y recibe mensajes desde tus propios sistemas.

### Chat externo (salida más rápida)

El **chat externo** te da una URL compartible para lanzar en minutos (en consola puede mostrarse como *External chat*).

<Steps>
  <Step title="Ve a Conecta > Canales">
    Abre tu agente y entra a la página **Conecta**.
  </Step>
  <Step title="Abre el chat externo">
    Selecciona la tarjeta de chat externo / *External chat*.
  </Step>
  <Step title="Copia la URL pública">
    En la pestaña **Domain setup**, busca la tarjeta **Default URL** y haz clic en el botón de copiar para copiar el enlace generado.
  </Step>
  <Step title="Prueba como usuario final">
    Abre el enlace en una ventana privada y valida casos de uso reales.
  </Step>
</Steps>

<Tip>
  También puedes conectar un **dominio personalizado** para que los usuarios accedan a tu Chat Externo desde una URL de marca como `chat.ejemplo.com/ai-agent`. Consulta la página de [Chat Externo](/channels/es/web) para ver todos los detalles de la pestaña de configuración de dominio.
</Tip>

### Burbuja web (embebido en tu sitio)

Usa la **burbuja web** cuando quieras el agente dentro de tu propia web (en consola puede aparecer como *Web bubble* / *Bubble*).

<Steps>
  <Step title="Abre el canal de burbuja">
    En **Conecta > Canales**, abre la tarjeta de burbuja web.
  </Step>
  <Step title="Genera credenciales y fragmentos de código">
    La consola muestra el fragmento (*snippet*) y las claves para la integración.
  </Step>
  <Step title="Añade el fragmento al sitio">
    Pega el script donde tu equipo implementa el código web.
  </Step>
  <Step title="Personaliza la experiencia">
    Ajusta marca, botón de apertura y mensaje de bienvenida.
  </Step>
</Steps>

### Canales de Meta (WhatsApp, Messenger, Instagram)

La página de Conecta siempre muestra una sección de **Meta** con tres tarjetas de canal:

| Canal | Descripción |
|---|---|
| **WhatsApp** | Canal de WhatsApp Business vía Meta |
| **Messenger** | Canal de Facebook Messenger vía Meta |
| **Instagram** | Canal de mensajería de Instagram vía Meta |

Cada tarjeta muestra un botón **Connect** que abre el flujo de configuración de ese canal específico. No necesitas configurar la integración de Meta primero — las tarjetas siempre están visibles para que puedas iniciar el proceso de conexión en cualquier momento.

#### Distribución: Enlace para compartir y Burbuja para incrustar

Una vez que un canal de Meta está conectado, la página de configuración del canal incluye una sección de **Distribución** titulada **"Cómo funciona"**. Esta sección te permite compartir el canal con los usuarios de dos formas, seleccionables mediante un botón tipo píldora en la parte superior:

##### Enlace para compartir

Genera una URL pública que abre el mismo flujo de chat que la burbuja embebida. Puedes:

- **Agregar parámetros clave/valor** — usa el editor en el lado izquierdo para agregar parámetros personalizados (ej. `plan: pro`). Cada parámetro se añade a la URL como query string y se guarda automáticamente en el `source_params` del usuario final.
- **Copiar el enlace** — la URL se actualiza en tiempo real en el lado derecho a medida que agregas o quitas parámetros. Haz clic en **Copiar** para copiarlo al portapapeles.
- **Abrir el enlace** — haz clic en **Abrir** para probar el enlace en una pestaña nueva.
- **Vista previa de source_params** — cuando tienes al menos un parámetro, se muestra exactamente qué se guardará en el registro del usuario final.

<Tip>
  Usa el enlace para compartir en correos electrónicos, anuncios, códigos QR o cualquier lugar donde quieras dirigir tráfico al chat. Los parámetros del query string se capturan automáticamente, así que puedes etiquetar visitantes por campaña, fuente o cualquier dimensión personalizada.
</Tip>

##### Burbuja para incrustar

Pega un fragmento de JavaScript en tu sitio web para mostrar una burbuja de chat flotante. El fragmento se muestra en una tarjeta de código con resaltado de sintaxis en el lado derecho. Usa el desplegable en el encabezado de la tarjeta para elegir entre tres variantes de fragmento:

| Variante | Descripción |
|---|---|
| **Estático** | Etiqueta a cada visitante con parámetros clave/valor fijos que defines en el editor del lado izquierdo. Los parámetros del query string de la página también se capturan automáticamente. |
| **Desde la URL** | Reenvía automáticamente los parámetros `?query=string` de la página. Usa el editor para definir parámetros de ejemplo y ver una vista previa de la URL resultante y los `source_params`. |
| **Desde tu app** | Toma valores de tu propia aplicación — objeto de usuario, dataLayer, etc. El fragmento incluye comentarios de ejemplo que puedes personalizar. |

Haz clic en **Copiar** en el encabezado de la tarjeta de código para copiar el fragmento al portapapeles.

<Note>
  El fragmento de inserción y el enlace para compartir siempre apuntan al host de producción (`https://consola.semantiks.com`), por lo que funcionan correctamente incluso cuando estás desarrollando en local.
</Note>

<Tip>
  Cualquier parámetro que adjuntes — ya sea mediante el enlace para compartir, el fragmento estático o el reenvío de URL — se guarda en el `source_params` del usuario final. Asegúrate de agregar las claves relevantes a la **lista de permitidos** en la tarjeta de Source Params debajo de la sección de Distribución para que se persistan.
</Tip>
