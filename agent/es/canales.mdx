  Un agente bien configurado da mejores respuestas desde la primera conversación.
</Info>

## Conecta tu agente con usuarios reales

En la consola, **Conectar** se organiza en 3 secciones:

- **Channels**: publica tu agente en canales propios y externos (Web Bubble, External Chat, WhatsApp y otros)
- **Integrations**: conecta herramientas/servicios externos de tu operación
- **API**: envía y recibe mensajes desde tus propios sistemas

Este paso convierte tu agente de "configurado" a "en producción".

## 1) Channels

Usa Channels cuando quieres que los usuarios hablen con tu agente desde superficies de chat.

### External Chat (salida más rápida)

**External Chat** te da una URL compartible para lanzar en minutos.

<Steps>
  <Step title="Ve a Conectar > Channels">
    Abre tu agente y entra a la página de Connect.
  </Step>
  <Step title="Abre External Chat">
    Selecciona la tarjeta de External Chat.
  </Step>
  <Step title="Copia la URL pública">
    Haz clic en compartir y copia el enlace generado.
  </Step>
  <Step title="Prueba como usuario final">
    Abre el link en incógnito y valida casos de uso reales.
  </Step>
</Steps>

### Web Bubble (embebido en sitio web)

Usa **Web Bubble** cuando quieres tener el agente dentro de tu propia web.

<Steps>
  <Step title="Abre el canal Bubble">
    En Connect > Channels, abre la tarjeta Bubble.
  </Step>
  <Step title="Genera credenciales/snippets">
    La consola te muestra el snippet y claves para la integración.
  </Step>
  <Step title="Agrega el snippet al sitio">
    Pega el script donde tu equipo implementa el código web.
  </Step>
  <Step title="Personaliza la experiencia">
    Ajusta branding, launcher y comportamiento de bienvenida.
  </Step>
</Steps>

### Proveedores externos (WhatsApp y otros)

El mismo flujo de Connect soporta proveedores externos a través de canales basados en integración.

<Tip>
  Para configuración detallada por canal, usa las guías específicas en `channels/es/*`.
</Tip>

## 2) Integrations

Usa Integrations para conectar sistemas externos de tu operación (por ejemplo, proveedores de mensajería, CRM/helpdesk o servicios personalizados).

Desde el punto de vista del usuario:

- Activas la integración una vez por organización.
- Luego configuras canales por agente.
- Si aplica, activas addons opcionales por canal.

Esto sigue el modelo del backend (integración de organización -> canal del agente -> addon opcional), pero en una experiencia guiada desde consola.

## 3) API

Usa API cuando tu backend o aplicación necesita enviar mensajes al agente de forma programática.

En Connect > API puedes crear canales API y usar la referencia que aparece en la consola.

Flujo típico:

<Steps>
  <Step title="Crea un API Channel">
    En Connect, haz clic en **Create API Channel**.
  </Step>
  <Step title="Copia identificadores requeridos">
    Obtén `agent_id`, identificador del canal y tu API key.
  </Step>
  <Step title="Envía una solicitud">
    Llama al endpoint de chat desde tu backend con contexto de usuario y payload.
  </Step>
  <Step title="Procesa la respuesta">
    Guarda identificadores de conversación y renderiza la respuesta del agente.
  </Step>
</Steps>

<Warning>
  Mantén las API keys siempre del lado servidor. No expongas secretos en frontend público.
</Warning>

## Orden recomendado de despliegue

1. Lanza con **External Chat** para validar rápido.
2. Integra **Web Bubble** en tu propiedad principal.
3. Agrega **integraciones externas** para canales operativos.
4. Migra a **API** cuando necesites experiencias producto más personalizadas.
