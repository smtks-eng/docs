## Conecta tu agente con usuarios reales

En la consola, **Conecta** se organiza en tres bloques:

- **Canales** — publica tu agente en canales propios y externos (burbuja web, chat externo, WhatsApp y otros; en la interfaz algunos nombres pueden aparecer en inglés).
- **Integraciones** — conecta herramientas o servicios externos de tu operación.
- **API** — envía y recibe mensajes desde tus propios sistemas.

Este paso convierte tu agente de «configurado» a «en producción».

## 1) Canales

Usa **Canales** cuando quieras que los usuarios hablen con tu agente desde superficies de chat.

### Chat externo (salida más rápida)

El **chat externo** te da una URL compartible para lanzar en minutos (en consola puede mostrarse como *External chat*).

<Steps>
  <Step title="Ve a Conecta > Canales">
    Abre tu agente y entra a la página **Conecta**.
  </Step>
  <Step title="Abre el chat externo">
    Selecciona la tarjeta de chat externo / *External chat*.
  </Step>
  <Step title="Copia la URL pública">
    Usa la opción para compartir y copia el enlace generado.
  </Step>
  <Step title="Prueba como usuario final">
    Abre el enlace en una ventana privada y valida casos de uso reales.
  </Step>
</Steps>

### Burbuja web (embebido en tu sitio)

Usa la **burbuja web** cuando quieras el agente dentro de tu propia web (en consola puede aparecer como *Web bubble* / *Bubble*).

<Steps>
  <Step title="Abre el canal de burbuja">
    En **Conecta > Canales**, abre la tarjeta de burbuja web.
  </Step>
  <Step title="Genera credenciales y fragmentos de código">
    La consola muestra el fragmento (*snippet*) y las claves para la integración.
  </Step>
  <Step title="Añade el fragmento al sitio">
    Pega el script donde tu equipo implementa el código web.
  </Step>
  <Step title="Personaliza la experiencia">
    Ajusta marca, botón de apertura y mensaje de bienvenida.
  </Step>
</Steps>

### Canales de Meta (WhatsApp, Messenger, Instagram)

La página de Conecta siempre muestra una sección de **Meta** con tres tarjetas de canal:

| Canal | Descripción |
|---|---|
| **WhatsApp** | Canal de WhatsApp Business vía Meta |
| **Messenger** | Canal de Facebook Messenger vía Meta |
| **Instagram** | Canal de mensajería de Instagram vía Meta |

Cada tarjeta muestra un botón **Conectar** que abre el flujo de configuración de ese canal específico. No necesitas configurar la integración de Meta primero — las tarjetas siempre están visibles para que puedas iniciar el proceso de conexión en cualquier momento.

Una vez que hayas conectado una o más cuentas para un tipo de canal, aparece un botón **Canales (n)** en la tarjeta mostrando el número de conexiones activas. Haz clic en él para ver, editar o reconectar tus conexiones existentes.

<Steps>
  <Step title="Abre la sección de Meta">
    Desplázate hasta la sección **Meta** en la página de Conecta.
  </Step>
  <Step title="Haz clic en Conectar en el canal deseado">
    Elige **WhatsApp**, **Messenger** o **Instagram** y haz clic en el botón **+&nbsp;Conectar**.
  </Step>
  <Step title="Completa el flujo de registro de Meta">
    Sigue el modal de registro integrado para autenticarte con tu cuenta de Meta Business y seleccionar el activo (número de teléfono, página o cuenta de Instagram) a conectar.
  </Step>
  <Step title="Verifica la conexión">
    Después de conectar, la tarjeta se actualiza para mostrar la cantidad de canales activos. Haz clic en el botón del conteo para revisar los detalles.
  </Step>
</Steps>

<Tip>
  Para una guía detallada paso a paso de configuración de WhatsApp, consulta la guía de integración [Cómo integrar WhatsApp](/integration-guides/es/como-integrar-whatsapp).
</Tip>

## 2) Integraciones

Usa **Integraciones** para conectar sistemas externos de tu operación (por ejemplo, proveedores de mensajería, CRM, mesa de ayuda o servicios propios).

Desde la perspectiva del usuario:

- Activas la integración una vez por organización.
- Luego configuras canales por agente.
- Si aplica, activas complementos (*addons*) opcionales por canal.

El modelo en backend es: integración de organización → canal del agente → complemento opcional, pero la consola lo presenta en un flujo guiado.

## 3) API

Usa **API** cuando tu propio backend o aplicación necesite enviar mensajes al agente de forma programática.

En **Conecta > API** puedes crear canales API y usar la referencia de API que se muestra en la consola.

<Steps>
  <Step title="Crea un canal API">
    Ve a Conecta > API y crea un nuevo canal API.
  </Step>
  <Step title="Copia las credenciales">
    Copia la clave API y el endpoint proporcionados.
  </Step>
  <Step title="Integra en tu sistema">
    Usa las credenciales y la referencia de API para enviar y recibir mensajes.
  </Step>
</Steps>

<Note>
  Consulta la referencia de API en la consola para la lista completa de endpoints y formatos de payload disponibles.
</Note>
