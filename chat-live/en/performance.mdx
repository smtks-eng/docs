The **Contact Center Performance** dashboard gives you a live and historical view of how your human-support operation is running. It combines real-time indicators (who is online, how many customers are waiting) with windowed KPIs (escalation rates, wait times, resolution times) so you can react quickly and plan strategically.

## Live Section

At the top of the dashboard you will find real-time metrics that refresh every 30 seconds:

| Metric | Description |
|---|---|
| **Available agents** | Number of human agents currently online and available to take conversations. |
| **Customers waiting** | Users currently in the queue waiting for a human agent. |
| **Active conversations** | Conversations that are actively being handled by a human agent right now. |
| **Longest wait** | The elapsed time of the oldest conversation still waiting in the queue. |

### Agent Roster

If your role is **Owner**, **Admin**, or **Member**, the Available agents card expands to show a **roster** of all connected team members. Each member is displayed with:

- A **green dot** and dark name if they are **available** (online and ready to take conversations).
- A **gray dot** and muted name if they are **connected but unavailable** (online but not accepting new conversations).

The count is displayed as **X of N**, where X is the number of available agents and N is the total number of connected members. The roster scrolls when the list is long, and the backend sorts available members first, then alphabetically.

<Tip>
Use the roster to quickly see who on your team is ready to handle escalations and who might need to toggle their availability.
</Tip>

## Windowed KPIs

Below the live section, you can select a time window (**Today, 7D, 30D, Year**) to see historical KPIs:

| KPI | Description |
|---|---|
| **Wait time** | Average time from queue entry to first human response. |
| **Resolution time** | Average time from escalation to conversation close. |
| **Escalation rate** | Percentage of total conversations that were escalated to a human agent. |
| **Total escalated** | Absolute number of escalated conversations in the period. |
| **Peak day** | The day with the highest number of escalations. |

## Escalation Conversations & Origin Tables

Two expandable tables provide more detail:

- **Conversations** — lists individual escalated conversations with status, start time, channel, profile name, and message count.
- **Origin** — breaks down escalations by channel so you can see which channels generate the most human-support demand.

<Tip>
Combine the live roster with the Origin table to understand whether your current staffing matches the channels driving the most escalations.
</Tip>
