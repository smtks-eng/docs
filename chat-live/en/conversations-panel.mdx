The **Conversations Panel** (Inbox) is your central hub for monitoring every conversation your AI agent is handling. From here you can watch conversations in real time, take control when human intervention is needed, and keep track of conversation status across all channels.

<Note>
Live Chat is a **Beta** feature. Some capabilities may evolve as the feature matures.
</Note>

## Real-time updates

The Conversations Panel uses **Server-Sent Events (SSE)** to deliver messages in real time. You no longer need to refresh the page or wait for polling intervals:

- **New messages** from users and the AI agent appear instantly in the conversation detail view.
- **Conversation list** updates automatically — the most recently active conversation bubbles to the top.
- **Status changes** (e.g. escalation, end of conversation) are reflected immediately in the sidebar list.

When you send a message as a human agent, it appears in the chat **optimistically** — meaning it shows up right away while the server confirms delivery in the background. If sending fails, the message is automatically removed.

## Conversation list

The left sidebar shows all conversations for your agent. Each item displays:

- **User name**
- **Last message** preview
- **Relative timestamp** (e.g. "5m", "2h", "3d", or a date for older conversations)
- **Status indicators** for active, escalated, or ended conversations

## Filtering conversations

Above the conversation list you'll find several filters to narrow down what you see:

| Filter | Description |
|---|---|
| **Search** | Free-text search by conversation summary or user name |
| **Channel** | Filter by a specific connected channel (dynamically populated from your enabled integrations) |
| **Assigned** | Filter by the team member a conversation is assigned to |
| **Escalation** | Toggle between **All**, **Escalated** (human-handled), or **Not escalated** (agent-handled) |
| **Status** | Filter by **Active** or **Inactive** (ended) conversations |
| **Sort** | Toggle between **Newest first** and **Oldest first** ordering |

<Tip>
Use the **Escalated** filter to quickly find conversations that need human attention, then combine it with the **Assigned** filter to see only the ones assigned to you.
</Tip>

## Conversation detail

Select a conversation from the list to open it. The detail view includes:

### Header

- **User name** — the contact's display name
- **Conversation history** — if this user has had previous conversations, a dropdown lets you navigate between them
- **Assigned to** — shows which team member is currently handling the conversation (if escalated)
- **Channel badge** — indicates which channel (e.g. WhatsApp, Web) the conversation is on
- **Status controls** — change the conversation status or take/release human control

### Message thread

The full message history between the user, the AI agent, and any human agents. Messages are labeled by sender.

### Sending messages

When you have taken control of a conversation, a message input appears at the bottom. Messages you send show up instantly (optimistic send) and are confirmed by the server in the background.

## Taking and releasing control

You can **take control** of any active conversation to respond as a human agent. The AI agent will pause its automatic replies while you handle the conversation.

To release control back to the AI agent, click the **Release** button. A status message is recorded in the conversation automatically.

## Customer details panel

Toggle the right-side panel to see detailed information about the contact, including their profile data and any memories collected by the agent.

## Key notes

- Real-time updates are powered by SSE — no manual refresh needed.
- The channel filter dynamically reflects only the channels you have connected and enabled for your agent.
- Conversations where no team member has interacted still show the AI agent's responses in real time.
- Sorting and filtering are handled server-side for consistent performance across large conversation volumes.
